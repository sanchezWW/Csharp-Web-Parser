# Коммерческий парсер игровых маркетплейсов (C# / Playwright)

Высокопроизводительный асинхронный парсер для мониторинга цен и лотов на игровых площадках (на примере FunPay). Спроектирован с учетом гибкой настройки через конфигурационные файлы и CLI-интерфейс.

## Стек технологий и библиотеки
* **Язык/Платформа:** C# / .NET 8
* **Парсинг и Автоматизация:** Playwright (эмуляция реального браузера, обход динамического JS)
* **Работа с данными:** EPPlus (генерация профессиональных Excel-отчетов `.xlsx`)
* **Интерфейс командной строки:** System.CommandLine

## Архитектура и паттерны (SOLID / Clean Code)
* **Принцип единственной ответственности (SRP):** Логика конфигурации, парсинга площадок и экспорта строго разделена по сервисам (`ScrapeService`, `ExcelExportService`).
* **Планы по развитию (Паттерн Strategy):** Архитектура подготавливается под паттерн "Стратегия" для легкого добавления новых маркетплейсов (выделение интерфейса парсера) без изменения основного цикла программы.
* **Гибкая конфигурация:** Использование `config.json` для управления прокси-серверами, таймаутами (`DelayBetweenRequestsSeconds`), списком отслеживаемых игр и режимом отображения браузера.

## Примеры использования (CLI)

# Стандартный запуск (считывает базовый config.json)
dotnet run

# Запуск в headless-режиме (без графического окна браузера для серверов/планировщиков)
dotnet run -- --headless

# Фильтрация по конкретной игре с кастомной задержкой
dotnet run -- --headless --delay 5 --game "Counter-Strike"

# Использование альтернативного файла конфигурации и вывода
dotnet run -- --config prod-config.json --output result.xlsx

# Вызов встроенной интерактивной справки
dotnet run -- --help
